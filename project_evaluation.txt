========================================================================
2. ADVANTAGES OF THE PROJECT
========================================================================

- Comprehensive End-to-End ML Pipeline: Seamlessly merges data extraction methodologies with multi-level modeling (return prediction, volatility filtering, optimization).
- Explainable AI (XAI): Integrates SHAP value generation so users understand *why* the algorithm selects a specific asset allocation, mitigating the "black-box" skepticism standard in ML trading apps.
- Dynamic Risk Management: Utilizing a Hidden Markov Model (HMM) for market regime detection (Bull vs Bear states) enables dynamic adjusting of portfolio risk tolerance depending on macro market health.
- Convex Portfolio Optimization: Goes beyond mere score prediction to construct a structurally resilient portfolio with mathematical constraints (long-only, max position caps, minimum weight limits) mapped directly to user risk tolerance levels.
- Modern User Interaction: A dynamic, aesthetically clean Next.js React frontend visualizes all critical decision nodes generated by the backend engine (from covariance matrices to sentiment analysis).


========================================================================
3. LIMITATIONS OF THE PROJECT
========================================================================

- Marginal Edge Against Transaction Costs: Expected hit rates hang narrowly above random probability (around 52%). In high-frequency or medium-frequency execution, real-world constraints such as brokerage fees, taxation, and slippage might erode predictive margins.
- Reliance on Free Data Tiers: Depending directly on Yahoo Finance restricts the depth of feature possibilities (e.g. limit order book data, granular alternative datasets) and could experience API throttling.
- Restricted Action Space (Long-Only): The optimizer is constrained exclusively to long positions without the capacity to deploy short-selling or derivates (Options/Futures) hedging which restricts true market-neutral protection during acute drawdowns.
- Reaction to Black Swan Events: Unforeseen structural shocks (e.g. sudden geopolitical crisis) inherently lag inside the Volatility models (GARCH) and the HMM state detector until they print in closing price histories. 
- Separation from Execution Brokers: The current deployment identifies ideal target weights theoretically but lacks direct API integrations (like Zerodha Kite Connect or Upstox) to automatically place those rebalancing trades.
